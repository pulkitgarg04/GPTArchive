# Contributing to GPTArchive

Thank you for your interest in contributing to GPTArchive! We welcome contributions from the community to help improve this project. Whether you're fixing a bug, improving documentation, or proposing a new feature, your efforts are appreciated.

## How to Contribute

### Reporting Bugs
If you encounter any issues while using GPTArchive, please open an issue on our [GitHub repository](https://github.com/pulkitgarg04/GPTArchive/issues). When reporting a bug, please include:
-   A clear and descriptive title.
-   Steps to reproduce the issue.
-   Expected behavior vs. actual behavior.
-   Screenshots or error logs, if applicable.
-   Your browser and OS version.

### Suggesting Enhancements
We are always looking for ways to improve. If you have an idea for a new feature or improvement, please open an issue and tag it as an "enhancement". Describe your idea in detail and explain how it would benefit users.

### Pull Requests
1.  **Fork the Repository**: Click the "Fork" button on the top right of the repository page.
2.  **Clone Your Fork**: Clone your forked repository to your local machine.
    ```bash
    git clone https://github.com/your-username/GPTArchive.git
    ```
3.  **Create a Branch**: Create a new branch for your feature or bug fix.
    ```bash
    git checkout -b feature/your-feature-name
    ```
4.  **Make Changes**: Implement your changes, ensuring you follow the project's coding standards.
5.  **Commit Changes**: Commit your changes with clear and descriptive messages.
    ```bash
    git commit -m "Add feature: description of changes"
    ```
6.  **Push to Your Fork**: Push your branch to your forked repository.
    ```bash
    git push origin feature/your-feature-name
    ```
7.  **Submit a Pull Request**: Go to the original repository and open a Pull Request (PR) from your forked branch. Provide a detailed description of your changes and reference any related issues.

## Code Style
-   Please ensure your code is clean, readable, and well-documented.
-   Follow standard JavaScript/HTML/CSS best practices.
-   Ensure no sensitive information is hardcoded.
